# Session Shadow Methodology

The selected session never changes Field 1 production truth or the protected Power BI central path. The shadow layer reads completed/matured historical evidence, applies session/horizon residual adjustment with shrinkage toward global evidence, caps changes using ATR constraints, and falls back from session+horizon to broader evidence and finally the unchanged central path. Manual selector changes invalidate only shadow presentation/cache state.

Published diagnostics include central/shadow price, adjustment, interval, sample count, weight, evidence tier, direction accuracy, fallback reason, and cap status where available. No improved trading-accuracy claim is made without matured out-of-sample evidence.
