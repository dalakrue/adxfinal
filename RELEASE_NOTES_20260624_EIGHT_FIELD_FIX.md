# Release Notes — Eight-Field Lunch Fix

## Corrected defects

- Restored Lunch Fields 7 and 8 as independent top-level closed-first fields.
- Removed Field 7/8 ownership from Field 6, preventing nested hiding and accidental loading.
- Added the missing `central_copy_result` implementation that had caused the copy renderer import to fail.
- Removed the duplicate top-copy renderer owner; one authoritative copy control remains.
- Extended structured copy and full/short payloads to include Fields 7 and 8.
- Updated visible Lunch architecture labels from six fields to eight fields.

## Performance-safe changes

- Fields 7 and 8 import/render only when individually opened.
- Copy payload materialization remains user-triggered; duplicate serialization was removed.
- No heavy calculation was added to Lunch.
- Generated bytecode caches, pytest cache, nested source backup ZIP and obsolete database backup are excluded from the release package.
- Protected production calculation formulas and Field 1 logic were not intentionally changed.

## Verification

- Python compile validation passed for modified modules.
- Automated suite: 450 tests passed (211 + 239 split batches).
- Headless Streamlit startup smoke test passed.
- Tests cover independent eight-field architecture, copy payload inclusion, Field 8 identity/publication behavior, protected imports and no heavy Lunch recalculation.

## Performance claim boundary

The release package size reduction is measured during packaging. The lazy-render and duplicate-work changes are expected to lower Lunch render-path CPU/RAM, but an exact 30–50% production runtime reduction is not claimed without a representative live-data benchmark on the deployment machine.
