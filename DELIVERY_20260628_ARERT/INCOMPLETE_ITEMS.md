# Explicit Incomplete Items

The software is repaired and deployable, but the academic framework is not represented as a fully validated thesis conclusion.

1. A fully fitted parametric Hidden Semi-Markov Model is not implemented; Module 1 uses empirical duration/survival behavior.
2. Complete Bayesian online changepoint posterior recursion is not implemented; Module 3 is clearly labelled as a robust point-in-time proxy.
3. Regime/session-specific conformal calibration falls back to pooled residuals when group samples are too small.
4. The meta-label model is an empirical calibrated prototype, not a final nested-walk-forward trained classifier.
5. Dynamic Bayesian model averaging is approximated by normalized inverse recent loss.
6. Prospect-theory loss-aversion selection is not finalized through an untouched holdout.
7. Event-response outputs are associative, not causal.
8. Complete Deflated Sharpe Ratio, White’s Reality Check, and SPA implementations are not finished.
9. ARERT currently uses an equal-weight baseline; walk-forward fitted weights are explicitly incomplete.
10. A physical iPhone/Android benchmark was not available. The measured 91.86% reduction applies to repeated same-candle ARERT computation on the test environment, not total phone RAM/CPU.
11. Real-key live API validation was not performed in the delivered environment. Mocked OpenRouter tests and safe fallbacks passed.
12. Thesis hypotheses H1–H7 still require sufficient real settled data and final holdout evaluation before acceptance or rejection.

These items are shown in module statuses, documentation, and UI limitations rather than being hidden or replaced with fabricated values.
